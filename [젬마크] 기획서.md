## 2. 기획 의도

### 2.1 문제 정의

gem.cafe를 통해 생성된 광고 영상은 소상공인의 창작물이자 비즈니스 자산이다. 그러나 디지털 영상은 복사·배포가 용이하여 다음과 같은 문제가 발생할 수 있다.

| 문제 | 내용 |
|---|---|
| 출처 불분명 | 온라인에 유통 중인 영상이 어느 사용자로부터 생성·배포된 것인지 확인 불가 |
| 무단 사용 탐지 불가 | 영상이 도용되어 타 계정에서 업로드되더라도 인지하기 어려움 |
| 법적 증빙 부재 | 저작권 분쟁 발생 시 원본임을 증명할 객관적 자료 부족 |

### 2.2 서비스 방향

gem.mark는 gem.cafe에서 다운로드되는 **모든 영상에 비가시적 포렌식 워터마크를 자동 삽입**하고, 이를 추출·검증·분석하는 보안 관리 플랫폼이다.

- 워터마크에는 영상 고유 UUID와 다운로드 사용자 ID가 인코딩되어, 무단 배포 영상 발견 시 원본 출처 및 최초 다운로더를 역추적할 수 있다.
- DCT 스프레드 스펙트럼 방식으로 삽입하여 육안으로 식별할 수 없으며, 영상 화질에 영향을 최소화한다.
- 압축·노이즈·크롭 등 다양한 공격에 대한 워터마크 강건성을 정기적으로 검증하고 BER·PSNR 지표로 보안 수준을 수치화한다.

## 3. 주요 기능

### 3.1 대시보드

운영 현황을 실시간으로 파악하는 메인 화면이다.

| 구성 요소 | 내용 |
|---|---|
| KPI 카드 | 누적 워터마크 영상 수, 평균 처리 시간, 평균 BER, 평균 PSNR |
| 검증 트렌드 차트 | 기간별 워터마크 검출 이력 시각화 |
| 공격 유형 통계 | 강건성 테스트에서 시도된 공격 유형별 분포 |
| 최근 활동 | 삽입·검출·강건성 테스트 최신 작업 목록 |

### 3.2 워터마크 삽입

**DCT 스프레드 스펙트럼** 방식으로 영상 Y채널 8×8 DCT 블록의 [2,1] 계수에 32비트 페이로드를 삽입한다.

**gem.cafe 연동 자동 삽입 흐름**

```
gem.cafe 사용자 다운로드 요청
  → gem.cafe BE: POST /watermark/embed-from-path 호출 (내부 HTTP)
  → gem.mark BE: DCT 워터마크 삽입 (video_uuid + downloader_user_id 해시 기반 페이로드)
  → H.264 CRF 0 + yuv444p 무손실 인코딩으로 저장
  → 삽입 정보(watermark_hex, alpha, content_uuid) DB 기록
  → gem.cafe FE: 워터마크 삽입 완료 파일 다운로드
```

**어드민 직접 삽입 (gem.mark 단독 사용)**

1. 영상 파일 업로드 (드래그앤드롭 또는 파일 선택)
2. Alpha 파라미터 설정 (워터마크 강도 조절, 1~100)
3. 삽입 실행 → 완료 후 다운로드

### 3.3 워터마크 검출

의심 영상을 업로드하면 DCT 추출 후 DB 저장값과 BER을 비교하여 원본 여부를 판정한다.

| 판정 기준 | 결과 |
|---|---|
| BER < 0.1 | **VERIFIED** — 원본 판정. Content UUID, Business ID, 삽입일, BER 값 표시 |
| BER ≥ 0.1 | **NOT VERIFIED** — 원본 불일치 |

- 8 프레임 균등 샘플링 → 각 프레임 DCT 추출 → 다수결(majority vote)로 최종 비트 결정
- DB 내 전체 활성 워터마크 레코드 중 BER 최솟값 매칭

### 3.4 강건성 테스트

다양한 공격 시나리오에서 워터마크 생존 여부를 자동으로 검증한다.

| 기능 | 내용 |
|---|---|
| 대상 선택 | 날짜 범위 입력으로 대상 영상 필터링 (300ms 디바운스) |
| 테스트 실행 | BE 비동기 처리 (fire-and-forget). 실행 즉시 이력 페이지로 이동 |
| 결과 시각화 | 레이더 차트 — 공격 유형별 BER/PSNR 방사형 표시 |
| 드릴다운 | 실패 영상별 원인 분석 (공격 유형, 지표 상세) |

### 3.5 리포트 생성

워터마크 보안 분석 리포트를 PDF/XLSX 형식으로 생성한다.

- 기간·포함 섹션(추출 메타데이터, 레이더 차트, 공격 결과 테이블, 모델 신뢰도) 커스터마이징
- 실시간 레이아웃 미리보기 제공

## 6. 기대 효과

### 6.1 저작권 보호

| 항목 | 내용 |
|---|---|
| 전수 보호 | gem.cafe에서 다운로드되는 모든 영상에 워터마크 자동 적용 |
| 역추적 가능 | 무단 배포 영상 발견 시 원본 출처 및 최초 다운로더 식별 가능 |
| 법적 활용 | 저작권 분쟁 발생 시 삽입 정보(DB 기록)를 원본 증빙 자료로 제출 가능 |

### 6.2 보안 수준 정량화

| 지표 | 의미 | 목표값 |
|---|---|---|
| BER (Bit Error Rate) | 워터마크 검출 정확도 | 0.1 이하 |
| PSNR | 워터마크 삽입 후 화질 보존 수준 | 40dB 이상 |
| 강건성 성공률 | 공격 시나리오별 워터마크 생존율 | 측정 및 모니터링 |

### 6.3 운영 효율화

| 항목 | 내용 |
|---|---|
| 자동화 | 워터마크 삽입이 gem.cafe 다운로드 흐름에 통합되어 수작업 없이 대규모 처리 가능 |
| 모니터링 | 대시보드를 통한 전체 보안 현황 실시간 파악 |
| 감사 대응 | 리포트 자동 생성으로 보안 감사 자료 준비 간소화 |
